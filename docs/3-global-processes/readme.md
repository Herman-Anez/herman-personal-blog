# Global Processes

Esta sección documenta los flujos de negocio e ingeniería transversales que involucran a múltiples módulos de Herman's Personal Page, sirviendo como guías operativas e integradoras de la solución.

---

## 🎯 Objective

Registrar los flujos extremo a extremo para asegurar la coherencia técnica del equipo de desarrollo, detallando:

- Qué actor o evento inicia la acción.
- Qué módulos del sistema participan y coordinan tareas.
- Qué scripts y pipelines de integración continua intervienen.
- Qué invariantes de compilación y calidad se garantizan al completarse.

---

## 📦 Documented Processes

- **[Global Process 01: Publicación de Contenidos Técnicos (MDX)](./gp-01-content-publication.md)**  
  Guía operativa para la redacción de artículos y casos de estudio de proyectos, describiendo la validación de frontmatter, Biome formatting, Husky hooks y compilación estática offline.

- **[Global Process 02: Actualización e Integración de Traducciones (i18n)](./gp-02-i18n-update.md)**  
  Flujo transversal para añadir o modificar namespaces de idioma bilingües, validando la simetría de diccionarios JSON y previniendo errores visuales.
