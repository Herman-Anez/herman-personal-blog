# Global Processes

This section documents business flows that cross multiple bounded contexts and do not fully belong to a single module.

## Objective

Record end-to-end processes so the team can understand:

- which module initiates the flow
- which modules participate
- which events or integrations intervene
- where the failure or compensation points are
